# docs — Thinking Documents

This folder holds the "why we're building it, what to build, how to build it, and what we've decided." Source code lives in `../src/`.

## Structure

| Folder | What goes here | Files |
|---|---|---|
| `00-product/` | Product north star: why we build it, who it's for, experience principles | product-spec.md, experience-principles.md |
| `01-plan/` | Technical spec, implementation blueprint, kickoff prompt | dev-plan-v2.md, start-here.md |
| `03-decisions/` | Key decisions log (shared across design / engineering) | decision-log.md |
| `archive/` | Superseded older versions — reference only, do not build from | dev-plan-v1-superseded.md |

> The numbering leaves gaps on purpose (e.g. `02-` for user research) so later design / research work can slot in.

## Where to start reading

1. **`00-product/product-spec.md`** — understand "why" first (vision, scope, target users).
2. **`00-product/experience-principles.md`** — experience principles to stay focused during development (with research sources). Re-read before building and when stuck.
3. **`01-plan/dev-plan-v2.md`** — then "how" (the active technical blueprint).
4. **`01-plan/start-here.md`** — step-by-step kickoff prompt for Cursor / Claude Code.
